# 章节笔记模板

```markdown
# 第 <n> 章实践：<title>

## 来源与目标

- 教材来源：<link>
- 官方参考：<link>

说明本章要解决的问题和完成后的可观察能力。

## 必须掌握

列出 3 到 5 个概念；每个概念连接到本地示例代码。

## 运行

提供最小安装、环境变量和执行命令。不要包含真实密钥。

若本章核心能力依赖真实模型或 Agent 运行时行为，必须同时提供：

- 离线确定性验证命令（`pytest` 或无需 API Key 的脚本）
- 真实模型运行命令（通过 `.env` 或环境变量读取密钥与模型配置）

并说明两者各自验证什么能力。

## 常见失败模式

说明错误现象、根因与处理方式，至少覆盖工具边界或状态边界。

## 自检与练习

给出可验证的修改任务；优先要求先写测试或可观察的断言。
```
